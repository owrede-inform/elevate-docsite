## Overview

### When to Use Radio Buttons
- **Mutually exclusive choices**: When users must select exactly one option from a group
- **Preference settings**: Setting default options or user preferences
- **Configuration options**: Choosing between different configuration modes
- **Survey responses**: Single-answer survey questions and forms
- **Payment methods**: Selecting one payment option from available choices

### When NOT to Use Radio Buttons
- **Multiple selections**: Use checkboxes when users can select multiple options
- **Binary choices**: Use switches or single checkboxes for simple on/off choices
- **Many options**: Consider dropdowns or other components for large option sets
- **Optional selection**: When no selection is a valid choice, consider different patterns

## Design Principles

### Clarity Principle
**Radio button options should be clearly labeled and mutually exclusive.** Ambiguous or overlapping options confuse users about which choice to make.

### Grouping Principle
**Related radio buttons should be visually grouped and properly labeled.** Clear grouping helps users understand the relationship between options.

### Selection Principle
**Exactly one option should be selectable within each radio group.** Breaking this fundamental rule creates unpredictable interface behavior.

### Accessibility Principle
**Radio groups must be keyboard accessible with proper focus management.** Screen readers need proper group labeling and option announcements.

## Best Practices

### Organizing Radio Options

**Why this matters:** Logical organization helps users find their preferred choice quickly. Poor organization increases cognitive load and selection errors.

### Writing Clear Option Labels

**Why this matters:** Clear, distinct labels prevent user confusion and selection mistakes. Ambiguous labels lead to form abandonment and errors.

### Managing Radio Group State

**Why this matters:** Proper state management ensures predictable behavior and provides clear feedback about current selections and validation status.

### Radio Group Attributes

<PageTable>

| Attribute | Type | Default | Description |
|-----------|------|---------|-------------|
| `label` | string | - | Group label for accessibility |
| `name` | string | - | Form field name for all radios in group |
| `orientation` | string | vertical | Layout orientation: vertical, horizontal |
| `size` | string | medium | Size variant: small, medium, large |
| `required` | boolean | false | Marks group as required for validation |
| `error` | string | - | Error message to display |
| `success` | string | - | Success message to display |
| `disabled` | boolean | false | Disables entire radio group |

</PageTable>

### Radio Button Attributes

<PageTable>

| Attribute | Type | Default | Description |
|-----------|------|---------|-------------|
| `value` | string | - | Value submitted when radio is selected |
| `checked` | boolean | false | Sets radio as selected by default |
| `disabled` | boolean | false | Disables individual radio button |
| `name` | string | - | Form field name (usually set by radio-group) |

</PageTable>

### JavaScript Radio Control

```javascript
// Get radio group element
const radioGroup = document.querySelector('elvt-radio-group');

// Get selected value
const selectedValue = radioGroup.value;

// Set selected value
radioGroup.value = 'option2';

// Listen for selection changes
radioGroup.addEventListener('change', (event) => {
  console.log('Selected:', event.detail.value);
});

// Validate radio group
const isValid = radioGroup.checkValidity();

// Get all radio buttons in group
const radios = radioGroup.querySelectorAll('elvt-radio');
```

## Accessibility

### Essential Accessibility Features

**Proper grouping and labeling:**
- Radio groups use fieldset/legend or appropriate ARIA labeling
- Each radio button has a clear, descriptive label
- Group labels clearly describe the choice being made
- Related options are visually and semantically grouped

**Keyboard accessibility:**
- Arrow keys navigate between radio buttons within a group
- Tab moves between radio groups and other form elements
- Space key selects the focused radio button
- Selected radio buttons are clearly indicated to keyboard users

**Screen reader support:**
- Radio group labels are properly announced
- Individual radio labels are read when focused
- Selection state changes are announced
- Required status and validation messages are communicated

**Clear visual design:**
- Selected and unselected states are visually distinct
- Focus indicators meet accessibility contrast requirements
- Disabled states are clearly communicated visually
- Radio buttons maintain usable size for motor accessibility
