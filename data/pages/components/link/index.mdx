## Overview

Links serve as the connective tissue of digital experiences, guiding users through content and enabling smooth navigation. ELEVATE links maintain consistency with design system principles while offering flexibility for different contexts and use cases.

### When to Use Links
- **Navigation**: Moving between pages, sections, or external resources
- **Cross-references**: Connecting related content or documentation
- **Actions**: Triggering downloads, opening external tools, or launching applications
- **Content discovery**: Providing pathways to explore additional information

## Usage

### Link Targets
Control how links open and handle external navigation:

```html
<!-- Internal navigation (default) -->
<elvt-link href="/internal-page">Internal Link</elvt-link>

<!-- External link (opens in new tab) -->
<elvt-link href="https://external.com" target="_blank">
  External Link
</elvt-link>

<!-- Download link -->
<elvt-link href="/document.pdf" download>
  Download Document
</elvt-link>
```

### Best Practices

**DO:**
- Use descriptive link text that explains the destination
- Indicate external links with icons or visual cues
- Maintain consistent styling within the same context
- Provide sufficient contrast for accessibility
- Use proper semantic markup for screen readers

**DON'T:**
- Use "click here" or "read more" without context
- Underline text that isn't a link
- Make links too small for touch interactions
- Use color alone to indicate link status
- Break links across multiple lines unnecessarily

## Accessibility

Links are fundamental to web accessibility and must be properly implemented:

### Semantic Structure
- Links use proper `<a>` elements with valid `href` attributes
- Link text clearly describes the destination or purpose
- External links are indicated for screen reader users
- Download links specify file type and size when relevant

### Keyboard Navigation
- All links are keyboard accessible using Tab and Enter
- Focus indicators are clearly visible and meet contrast requirements
- Skip links help users navigate efficiently through content
- Tab order follows logical page structure

### Screen Reader Support
Links provide context and purpose through:
- Descriptive link text that makes sense out of context
- ARIA labels for complex interactions
- Proper announcement of link states and destinations
- Clear indication of external links and file downloads
