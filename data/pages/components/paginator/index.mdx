## Overview

### When to Use Paginators
- **Large datasets**: Breaking up long lists or tables into manageable pages
- **Search results**: Navigating through multiple pages of search results  
- **Product catalogs**: Browsing through extensive product listings
- **Data tables**: Managing large data tables with many rows
- **Content archives**: Navigating through chronological content like blogs or news

### When NOT to Use Paginators
- **Short lists**: Small datasets that fit comfortably on one page
- **Infinite scroll contexts**: Where continuous scrolling is more appropriate
- **Single-page content**: When all content should remain visible together
- **Real-time data**: Dynamic data where pagination boundaries would be unstable

## Design Principles

### Clarity Principle
**Pagination controls should clearly indicate current position and available navigation options.** Users need to understand where they are and how to move through the data.

### Efficiency Principle
**Pagination should minimize the number of clicks needed to reach desired content.** Efficient navigation patterns reduce user frustration and improve task completion.

### Consistency Principle
**Pagination behavior and appearance should be consistent across the application.** Predictable pagination patterns help users build mental models for navigation.

### Accessibility Principle
**Pagination controls must be fully accessible to keyboard and screen reader users.** All navigation options should be reachable and understandable through assistive technology.

## Best Practices

### Choosing Pagination Types

**Why this matters:** The right pagination type depends on dataset size and user needs. Simple navigation works for linear browsing, while numbered pagination enables direct access to specific pages.

### Managing Page Size

**Why this matters:** Appropriate page sizes balance loading performance with user convenience. Too few items per page creates excessive clicking, while too many items hurt performance.

### Providing Clear Feedback

**Why this matters:** Users need to understand their current position and available options. Clear pagination feedback prevents confusion and supports efficient navigation.

### Paginator Attributes

<PageTable>

| Attribute | Type | Default | Description |
|-----------|------|---------|-------------|
| `current-page` | number | 1 | Currently active page number |
| `total-pages` | number | - | Total number of available pages |
| `total-items` | number | - | Total number of items across all pages |
| `page-size` | number | 10 | Number of items displayed per page |
| `show-page-numbers` | boolean | false | Shows numbered page buttons |
| `max-visible-pages` | number | 5 | Maximum page numbers to show at once |
| `show-info` | boolean | true | Shows pagination information text |
| `info-format` | string | range | Information format: range, page, total |
| `variant` | string | default | Visual variant: default, simple, compact |
| `size` | string | medium | Size variant: small, medium, large |
| `disabled` | boolean | false | Disables all pagination controls |

</PageTable>

### JavaScript Paginator Control

```javascript
// Get paginator element
const paginator = document.querySelector('elvt-paginator');

// Listen for page changes
paginator.addEventListener('page-change', (event) => {
  const { currentPage, pageSize } = event.detail;
  console.log(`Navigate to page ${currentPage}, showing ${pageSize} items`);
  
  // Load new page data
  loadPageData(currentPage, pageSize);
});

// Update paginator programmatically
paginator.currentPage = 5;
paginator.totalPages = 25;
paginator.totalItems = 247;

// Navigate to specific page
paginator.goToPage(10);

// Navigate relative to current page
paginator.goToNextPage();
paginator.goToPreviousPage();
```

## Accessibility

### Essential Accessibility Features

**Keyboard accessibility:**
- All pagination controls are fully keyboard accessible
- Tab key moves through pagination controls in logical order
- Arrow keys can navigate between page numbers
- Enter and Space keys activate pagination buttons
- Focus indicators are clearly visible on all interactive elements

**Screen reader support:**
- Pagination controls use proper ARIA roles and labels
- Current page is clearly announced to screen readers
- Page changes are announced when navigation occurs
- Total pages and current position are communicated
- Disabled states are properly conveyed to assistive technology

**Clear navigation feedback:**
- Current page is visually distinct from other page options
- Disabled navigation buttons are clearly indicated
- Loading states are communicated during page transitions
- Error states provide clear feedback and recovery options

**Predictable behavior:**
- Pagination controls behave consistently across the application
- Navigation results match user expectations
- Page changes don't cause unexpected focus changes
- Pagination state is preserved during page updates
