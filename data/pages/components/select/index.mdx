## Overview

### When to Use Select
- **Limited screen space**: When radio buttons would take too much vertical space
- **Many options**: Lists with 7+ options that need to be scannable
- **Familiar patterns**: Standard form fields like country, state, category selection
- **Optional selection**: When users don't need to see all options immediately
- **Searchable content**: Large option lists that benefit from type-ahead filtering

### When NOT to Use Select
- **2-3 options**: Use Radio buttons for few mutually exclusive choices
- **Multiple selection**: Use Checkbox groups for selecting multiple items
- **Boolean choices**: Use Switch component for simple on/off states
- **Primary actions**: Use Button component for actions that trigger processes
- **Always visible options**: When users need to see all choices without interaction

## Design Principles

### Clarity Principle
**Options should be immediately understandable and mutually exclusive.** Users should be able to quickly scan options and understand what each choice means without additional explanation or context.

### Efficiency Principle  
**Select components should minimize the steps required to make a choice.** Well-organized options with logical ordering and search capabilities help users find their preferred option quickly.

### Predictability Principle
**Select behavior should match user expectations from other web forms.** Consistent interaction patterns and familiar option presentations reduce cognitive load and improve form completion rates.

### Scalability Principle
**Select components should work well with both small and large option sets.** Whether displaying 5 or 500 options, the component should maintain usability through appropriate features like search and grouping.

## Best Practices

### Writing Clear Option Labels

**Why this matters:** Ambiguous or inconsistent option labels force users to guess what they're selecting, leading to form errors and abandonment. Clear labels improve completion rates and user confidence.

### Common Usage Patterns

```html
<!-- Basic select -->
<elvt-select name="category" placeholder="Choose category">
  <label slot="label">Category</label>
  <elvt-option value="electronics">Electronics</elvt-option>
  <elvt-option value="clothing">Clothing</elvt-option>
  <elvt-option value="books">Books</elvt-option>
</elvt-select>

<!-- Select with default value -->
<elvt-select name="status" value="active">
  <label slot="label">Status</label>
  <elvt-option value="active">Active</elvt-option>
  <elvt-option value="inactive">Inactive</elvt-option>
  <elvt-option value="pending">Pending</elvt-option>
</elvt-select>

<!-- Required select with validation -->
<elvt-select name="priority" required>
  <label slot="label">Priority Level *</label>
  <span slot="help-text">Please select a priority level</span>
  <elvt-option value="">Select priority</elvt-option>
  <elvt-option value="low">Low</elvt-option>
  <elvt-option value="high">High</elvt-option>
  <elvt-option value="urgent">Urgent</elvt-option>
</elvt-select>

<!-- Searchable select for large lists -->
<elvt-select name="country" searchable placeholder="Search countries...">
  <label slot="label">Country</label>
  <elvt-option value="us">United States</elvt-option>
  <elvt-option value="ca">Canada</elvt-option>
  <elvt-option value="uk">United Kingdom</elvt-option>
  <!-- Many more options -->
</elvt-select>

<!-- Disabled select -->
<elvt-select name="department" disabled>
  <label slot="label">Department</label>
  <span slot="help-text">This field is not available for your role</span>
  <elvt-option value="engineering">Engineering</elvt-option>
  <elvt-option value="design">Design</elvt-option>
</elvt-select>
```

### Key Properties

<PageTable>

| Property | Options | Description |
|----------|---------|-------------|
| `name` | String | Form field name for data submission |
| `value` | String | Currently selected option value |
| `placeholder` | String | Placeholder text shown when no option selected |
| `required` | `true`, `false` (default) | Whether selection is required for form submission |
| `disabled` | `true`, `false` (default) | Whether the select can be interacted with |
| `searchable` | `true`, `false` (default) | Enable type-ahead search functionality |
| `clearable` | `true`, `false` (default) | Allow clearing the selected value |
| `size` | `small`, `medium` (default), `large` | Physical size of the select field |

</PageTable>

**Slots:** Use `slot="label"` for labels, `slot="help-text"` for helper text. Options are added as `<elvt-option>` child elements.

*For complete API reference including all properties, events, and methods, see the [Storybook documentation](https://elevate-core-ui.inform-cloud.io/?path=/docs/components-forms-select--docs).*

## Accessibility

### Essential Accessibility Practices

**Proper labeling is critical:**
```html
<!-- Good: Clear label association -->
<elvt-select name="country" required>
  <label slot="label">Country *</label>
  <span slot="help-text">Select your country of residence</span>
  <elvt-option value="">Choose a country</elvt-option>
  <elvt-option value="us">United States</elvt-option>
</elvt-select>

<!-- Bad: Missing or unclear labels -->
<elvt-select name="country">
  <elvt-option value="us">United States</elvt-option>
</elvt-select>
```

**Keyboard navigation works automatically:**
- Up/Down arrows navigate through options
- Enter or Space opens/closes the dropdown
- Escape closes dropdown without selection
- Tab moves focus to next form element
- Type-ahead search works when searchable is enabled

**Screen reader support:**
- Selected values are announced clearly
- Option lists are properly identified
- Required fields are communicated to assistive technology
- Search functionality is announced when available

**Focus management:**
- Clear focus indicators that meet WCAG contrast requirements
- Focus remains on select during option navigation
- Focus moves logically when dropdown opens/closes

**Provide meaningful option text:**
- Options should be self-explanatory without additional context
- Use consistent formatting across related selects
- Include helpful information in option labels when needed
