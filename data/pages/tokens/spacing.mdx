ELEVATE spacing system uses measure tokens for raw dimensional values and alias tokens for semantic spacing that creates consistent rhythm and hierarchy across all interface layouts and components.

</PageDescription>

## Spacing Architecture

### Measure Tokens
Raw dimensional values that form the foundation of the ELEVATE spacing system.

#### Base Spacing Scale
Built on a 4px base unit with harmonic progression:

```css
/* Base Spacing Measures */
--elvt-measures-spacing-3xs: 0.125rem  /* 2px */
--elvt-measures-spacing-2xs: 0.25rem   /* 4px */
--elvt-measures-spacing-xs: 0.5rem     /* 8px */
--elvt-measures-spacing-s: 0.75rem     /* 12px */
--elvt-measures-spacing-m: 1rem        /* 16px */
--elvt-measures-spacing-l: 1.5rem      /* 24px */
--elvt-measures-spacing-xl: 2rem       /* 32px */
--elvt-measures-spacing-2xl: 3rem      /* 48px */
--elvt-measures-spacing-3xl: 4rem      /* 64px */
--elvt-measures-spacing-4xl: 6rem      /* 96px */
--elvt-measures-spacing-5xl: 8rem      /* 128px */
```

#### Component Dimensions
Specific sizing values for common component dimensions:

```css

--elvt-measures-size-l: 2.5rem         /* 40px */
--elvt-measures-size-xl: 3rem          /* 48px */
--elvt-measures-size-2xl: 4rem         /* 64px */

/* Border Dimensions */
--elvt-measures-borderWidth-xs: 0.0625rem  /* 1px */
--elvt-measures-borderWidth-s: 0.125rem    /* 2px */
--elvt-measures-borderWidth-m: 0.25rem     /* 4px */
--elvt-measures-borderWidth-l: 0.5rem      /* 8px */

/* Border Radius */
--elvt-measures-borderRadius-xs: 0.125rem  /* 2px */
--elvt-measures-borderRadius-s: 0.25rem    /* 4px */
--elvt-measures-borderRadius-m: 0.5rem     /* 8px */
--elvt-measures-borderRadius-l: 0.75rem    /* 12px */

```

Semantic spacing tokens that reference measures for layout purposes.

```css

--elvt-alias-layout-content-spacing-l: {measures.spacing-2xl}    /* 48px */
```

## Usage Examples

### Using Spacing Tokens in CSS

```css
/* Use component tokens for specific components */
.my-button {
  padding: var(--elvt-component-button-padding-medium);
}

/* Use alias tokens for layout purposes */
.content-section {
  margin-bottom: var(--elvt-alias-layout-section-spacing-m);
  padding: var(--elvt-alias-layout-content-spacing-l);
}

/* Use measure tokens for custom spacing needs */
.custom-element {
  gap: var(--elvt-measures-spacing-m);
  border-radius: var(--elvt-measures-borderRadius-s);
}
```

### Using Spacing Tokens in SCSS

```scss
@import '@inform-elevate/elevate-design-tokens/dist/scss/tokens';

.layout {
  &__container {
    padding: var(--elvt-alias-layout-content-spacing-m);
    gap: var(--elvt-alias-layout-grid-gap-l);
  }
  
  &__section {
    margin-bottom: var(--elvt-alias-layout-section-spacing-l);
  }
  
  &__stack {
    display: flex;
    flex-direction: column;
    gap: var(--elvt-alias-layout-stack-spacing-m);
  }
  
  &__inline {
    display: flex;
    gap: var(--elvt-alias-layout-inline-spacing-s);
  }
}
```

### Creating Responsive Spacing

```css
/* Mobile-first responsive spacing */
.responsive-container {
  padding: var(--elvt-alias-layout-content-spacing-s);
}

@media (min-width: 768px) {
  .responsive-container {
    padding: var(--elvt-alias-layout-content-spacing-m);
  }
}

@media (min-width: 1024px) {
  .responsive-container {
    padding: var(--elvt-alias-layout-content-spacing-l);
  }
}
```

### Common Spacing Patterns

#### Content Hierarchy
```css
/* Article content spacing */
.article {
  h1 { margin-bottom: var(--elvt-measures-spacing-l); }
  h2 { margin: var(--elvt-measures-spacing-xl) 0 var(--elvt-measures-spacing-m); }
  h3 { margin: var(--elvt-measures-spacing-l) 0 var(--elvt-measures-spacing-s); }
  p { margin-bottom: var(--elvt-measures-spacing-m); }
  ul, ol { margin-bottom: var(--elvt-measures-spacing-m); }
  li { margin-bottom: var(--elvt-measures-spacing-xs); }
}
```

#### Form Layouts
```css
/* Form field spacing */
.form {
  &__field {
    margin-bottom: var(--elvt-component-field-group-spacing);
  }
  
  &__label {
    margin-bottom: var(--elvt-component-field-label-spacing);
  }
  
  &__help-text {
    margin-top: var(--elvt-component-field-help-text-spacing);
  }
  
  &__actions {
    margin-top: var(--elvt-alias-layout-section-spacing-s);

  }

```

#### Grid Layouts
```css
/* Card grid spacing */

}
```

## Best Practices

1. **Follow the hierarchy**: Component → Alias → Measure tokens
2. **Use consistent scales**: Stick to the defined spacing scale for harmony
3. **Consider touch targets**: Ensure interactive elements have adequate spacing
4. **Test responsively**: Verify spacing works across different screen sizes
5. **Maintain rhythm**: Use consistent spacing patterns for visual cohesion
6. **Group related elements**: Use smaller spacing for related items, larger for unrelated

## Accessibility

ELEVATE spacing tokens support accessibility by:

- **Touch targets**: Minimum 44px touch target sizes for interactive elements
- **Reading rhythm**: Appropriate line spacing and paragraph spacing for readability
- **Focus indicators**: Adequate spacing around focus outlines
- **Content separation**: Clear spacing between sections for screen reader navigation
- **Responsive scaling**: Spacing adapts appropriately across different zoom levels
