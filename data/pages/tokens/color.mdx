ELEVATE color system uses a 3-tier architecture where primitive colors form the foundation, aliases provide semantic meaning, and component tokens ensure consistent implementation across all interface elements.

</PageDescription>

## Color Architecture

Raw color values that form the foundation of the ELEVATE color system. These are context-agnostic and rarely change.

### Alias Tokens
Semantic color tokens that reference primitives and provide meaning-based naming.


## Usage Examples

### Using Color Tokens in CSS

```css
/* Use component tokens first */
.my-button {
  background-color: var(--elvt-component-button-background-color-primary);
  color: var(--elvt-component-button-text-color-primary);
}

/* Fall back to alias tokens for general purposes */
.my-text {
  color: var(--elvt-alias-content-text-color-default);
}

/* Use primitives only when no alias exists */
.custom-element {
  border-color: var(--elvt-primitives-color-blue-200);
}
```

### Using Color Tokens in SCSS

```scss
@import '@inform-elevate/elevate-design-tokens/dist/scss/tokens';

.alert {
  &--success {
    background-color: var(--elvt-alias-feedback-success-fill-default);

    color: var(--elvt-alias-feedback-success-text-default);
  }
  
  &--error {

    border: 1px solid var(--elvt-alias-feedback-error-border-default);
    color: var(--elvt-alias-feedback-error-text-default);
  }
}
```

## Theme Support

ELEVATE color tokens support light and dark themes through token value overrides:

### Light Theme (Default)
```scss
@import '@inform-elevate/elevate-design-tokens/dist/scss/light';
```

### Dark Theme
```scss
@import '@inform-elevate/elevate-design-tokens/dist/scss/dark';
```

In dark theme, alias tokens automatically map to appropriate dark primitives while maintaining the same semantic meaning.

## Accessibility

All ELEVATE color tokens are designed with accessibility in mind:

- **Contrast ratios** meet WCAG 2.1 AA standards (4.5:1 for normal text, 3:1 for large text)
- **Color blindness** considered with sufficient luminance differences
- **Focus indicators** use high-contrast colors for keyboard navigation
- **Status colors** don't rely solely on color but include additional visual cues

## Best Practices

1. **Follow the hierarchy**: Component → Alias → Primitive
2. **Use semantic naming**: Choose tokens that match your intent
3. **Test in both themes**: Verify colors work in light and dark modes
4. **Consider accessibility**: Ensure sufficient contrast ratios
5. **Document custom colors**: When extending the system, follow naming conventions

<InlineNotification>

**Remember:** Always prefer semantic alias tokens over primitives. This ensures your colors adapt properly across different themes and contexts.

</InlineNotification>
