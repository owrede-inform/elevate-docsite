# Color Tokens

<EsdsStatusBadge status="stable" />

Colors are fundamental to creating effective user interfaces. The ELEVATE color system provides a systematic approach to color usage, ensuring accessibility, consistency, and visual hierarchy across your applications.

## Color Architecture

ELEVATE uses a three-tier color token architecture that provides both consistency and flexibility:

1. **Primitive Tokens**: Base color values (immutable)
2. **Alias Tokens**: Semantic color meanings (immutable)  
3. **Component Tokens**: Context-specific usage (customizable)

<EsdsCodeBlock language="css">
```css
/* Primitive: Base color value */
--elvt-primitives-color-blue-500: #3b82f6;

/* Alias: Semantic meaning */
--elvt-alias-color-primary: var(--elvt-primitives-color-blue-500);

/* Component: Specific usage */
--esds-button-background: var(--elvt-alias-color-primary);
```
</EsdsCodeBlock>

### When to Use Status Colors

- **Success (Green)**: Confirmation messages, successful actions, positive states
- **Warning (Amber)**: Caution messages, non-critical warnings, attention needed
- **Danger (Red)**: Error messages, destructive actions, critical warnings  
- **Info (Blue)**: Informational messages, tips, neutral notifications

- **Gray 50-200**: Light backgrounds, subtle borders
- **Gray 300-400**: Medium contrast borders, disabled states  
- **Gray 500-600**: Secondary text, icons
- **Gray 700-900**: Primary text, high contrast elements

## Accessibility

All color combinations in ELEVATE meet WCAG AA accessibility standards:

- **Text contrast**: Minimum 4.5:1 ratio for normal text, 3:1 for large text
- **Interactive elements**: All buttons and links meet contrast requirements
- **Status colors**: Designed to be distinguishable for colorblind users
- **Focus indicators**: High contrast focus rings for keyboard navigation

<EsdsCodeBlock language="css">
```css
/* Good: High contrast text */
color: var(--elvt-alias-text-primary);
background: var(--elvt-alias-background-primary);

/* Good: Accessible link color */
color: var(--elvt-alias-text-accent);

/* Avoid: Low contrast combinations */
color: var(--elvt-primitives-color-gray-400);
background: var(--elvt-primitives-color-gray-50);
```
</EsdsCodeBlock>

## Implementation

### Using Color Tokens

<EsdsCodeBlock language="css">
```css
/* Use alias tokens for semantic meaning */
.button-primary {
  background-color: var(--elvt-alias-color-primary);
  color: var(--elvt-alias-text-inverse);
}

/* Use primitive tokens for specific shades */
.notification-background {
  background-color: var(--elvt-primitives-color-blue-50);
  border: 1px solid var(--elvt-primitives-color-blue-200);
}
```
</EsdsCodeBlock>

### Theme Customization

<EsdsCodeBlock language="css">
```css
/* Override alias tokens for custom themes */
:root {
  --elvt-alias-color-primary: #6366f1; /* Custom purple primary */
  --elvt-alias-color-secondary: #10b981; /* Custom green secondary */
}
```
</EsdsCodeBlock>

## Best Practices

### Do's
- Use semantic alias tokens instead of primitive values when possible
- Maintain consistent color usage across your application
- Test color combinations for accessibility compliance
- Use status colors appropriately to convey meaning

### Don'ts  
- Don't use primitive tokens directly for semantic purposes
- Don't create custom colors without considering accessibility
- Don't use color alone to convey important information
- Don't mix different color systems within the same interface
